## Problem

Given an array `nums`, return all unique triplets `[a, b, c]` such that
`a + b + c == 0`.

## Approaches

### Brute force

Check every triplet `(i, j, k)` and collect those summing to zero, deduplicating
with a set.

- Time: `O(n^3)`
- Space: `O(n)`

### Optimal

Sort the array. Fix each index `i`, then run an opposite-ends two-pointer scan
over the remaining suffix looking for `-nums[i]`. Move `left`/`right` based on the
running total and skip duplicate values so each triplet is reported once. Break
early once `nums[i] > 0`.

- Time: `O(n^2)`
- Space: `O(1)` (ignoring the output and sort)

## Key insight

Sorting reduces the third value to a classic two-sum-on-a-sorted-array problem,
and skipping equal neighbors gives uniqueness without a hash set.

## Edge cases

- Fewer than three elements.
- All zeros (single triplet `[0, 0, 0]`).
- Many duplicates that must not produce repeated triplets.

## Pitfalls

- Skipping duplicates *before* recording a valid triplet and missing an answer.
- Advancing only one pointer after finding a match, causing duplicates.
- Forgetting the outer duplicate skip on `nums[i]`.
