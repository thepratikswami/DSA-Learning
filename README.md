# DSA Learning

This repository is organized around core DSA patterns and contains solved
LeetCode-style Python problems for each pattern. It targets interviews at top
companies: every problem ships with a study note (approaches, complexity,
edge cases, curated company tags) and every topic has a revision sheet.

- `hashing`
- `two-pointers`
- `sliding-window`
- `prefix-sum`
- `binary-search`
- `stack`
- `monotonic-stack`
- `linked-list`
- `trees`
- `trie`
- `heap`
- `intervals`
- `greedy`
- `backtracking`
- `graphs`
- `advanced-graphs`
- `dynamic-programming`
- `bit-manipulation`
- `math-geometry`
- `design`

Each topic folder contains:

- `README.md`: a beginner-friendly overview of the pattern.
- `00-revision.md`: a quick interview revision sheet with a worked example.
- Named problem files with `Solution` implementations.
- A per-problem `.md` study note next to each solution (approaches, complexity,
  key insight, edge cases, pitfalls, and curated company tags).
- A small debug runner block in most problem files for quick local execution.

## Study System

A full interview-prep workflow lives in the [`study/`](study/) folder:

- [study/roadmap.md](study/roadmap.md): recommended topic order and 1/4/8-week plans.
- [study/pattern-flashcards.md](study/pattern-flashcards.md): map a new problem to a pattern in two minutes.
- [study/spaced-repetition.md](study/spaced-repetition.md): a review schedule so patterns stick.
- [study/mock-interview-checklist.md](study/mock-interview-checklist.md): grade your timed mocks.

## How To Use

1. Read [study/roadmap.md](study/roadmap.md) and pick a plan.
2. For each topic, start with its `README.md`, then `00-revision.md`.
3. Solve a problem yourself before reading its `.md` note.
4. Check the box in the Progress Tracker and schedule a review.
5. Run `python3 create_solution.py --progress` to see your coverage.

## Add New Problems

Create a new starter file whenever you want to solve a problem:

```bash
python3 create_solution.py
```

`create_solution.py` asks you to choose a topic folder, enter the LeetCode
problem number, problem title, solution method name, difficulty, and company
tags. It creates a starter file using the repository naming style
(`problem-number-kebab-case-title.py`), a matching `.md` study note, and
refreshes the Progress Tracker below.

Company tags are a curated, approximate "frequently asked at" list and change
over time; treat them as prioritization hints, not authoritative data.

Rebuild the tracker at any time without creating a problem:

```bash
python3 create_solution.py --reindex
```

Checkbox state you set in the tracker is preserved across reindexing. See your
completion per topic at any time:

```bash
python3 create_solution.py --progress
```
